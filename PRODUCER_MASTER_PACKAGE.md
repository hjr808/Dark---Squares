## 📚 CONTENT SPECIFICATIONS

### Book Format Standards

#### Manuscript Structure
```markdown
# Dark Squares — Book One
**Author:** Alfred HJR
**Status:** [Draft version]
**Word Count:** [Current/Target]

## Chapter 1
[Chapter title/description]

### Scene 1.1
[Content]

### Scene 1.2
[Content]

## Chapter 2
[And so on...]
```

#### Per-Chapter Guidelines
- **Target length:** 2,000-3,500 words per chapter
- **Scene structure:** 3-5 scenes per chapter
- **Elements:** Action, dialogue, internal monologue, description
- **Pacing:** Vary scene length for rhythm
- **Thematic elements:** Each chapter reinforces "dark squares" concept

#### Character Voice
- Street-authentic dialogue
- Internal monologue reflecting moral complexity
- Perspective focused on protagonist
- Supporting character scenes sparse but impactful

### Episode Format Standards (Future)

#### Script Structure
```
TEASER (5-7 pages)
ACT ONE (10-12 pages)
ACT TWO (10-12 pages)
ACT THREE (10-12 pages)
ACT FOUR (8-10 pages)

Total: ~45-50 pages
```

#### Episode Breakdown
- Title & logline
- Teaser/cold open
- Act breaks
- Character scenes
- Plot advancement
- Emotional beats

## 🔄 WORKFLOW & COLLABORATION

### File Update Process

1. **Create new content:**
   - Write in appropriate directory
   - Follow naming conventions
   - Include metadata header

2. **Draft & Review:**
   - Peer review (2-3 team members minimum)
   - Comment & suggest revisions
   - Track changes if using Word/Google Docs

3. **Finalize:**
   - Incorporate feedback
   - Update status in document
   - Mark as final version

4. **Commit to Repository:**
   - Add to GitHub with clear commit message
   - Link to Master Package if significant
   - Update Archive & Versioning if replacing document

5. **Archive Previous Version:**
   - Move old version to Archive folder
   - Update Asset Inventory spreadsheet
   - Document reason for replacement

### Naming Conventions

**Folders:**
- Use underscores for spaces: `Dark-Squares-Book-1/`
- Descriptive names: avoid generic "v1", "draft1"
- Consistent hierarchy

**Files:**
- Descriptive titles: `Dark-Squares-Book-1.md`
- Version in filename if needed: `Book-1_Draft-1.md`, `Book-1_Final.md`
- Include date if tracking iterations: `Book-1_072626.md`

**File Formats:**
- GitHub primary: `.md` (Markdown)
- Distribution: `.docx` (Word), `.pdf` (PDF)
- Design files: `.ai` (Adobe Illustrator), `.psd` (Photoshop)
- Source: `.xlsx` (Excel), `.csv` (CSV export)
